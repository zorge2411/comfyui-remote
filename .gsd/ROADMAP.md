### Phase 70: Subgraph Expansion

**Status**: üéØ Planned
**Objective**: Implement subgraph expansion to properly handle ComfyUI group nodes
**Depends on**: Phase 69

**Context**: ComfyUI workflows use subgraphs (group nodes) to encapsulate complex logic. These appear as UUID-typed nodes with `proxyWidgets` that reference internal nodes. Currently, these are preserved in the API JSON but rejected by the server. Solution is to expand subgraphs by replacing the wrapper node with its  internal nodes.

**Tasks**:

- [ ] Parse subgraph definitions from `definitions.subgraphs`
- [ ] Implement node ID remapping to avoid conflicts
- [ ] Remap internal links to connect to external graph
- [ ] Replace subgraph wrapper nodes with expanded internal nodes
- [ ] Verify with real subgraph workflow
